## 二、热力图密度渲染（任务 B）

### 目标
在三维场景上叠加 POI 密度热力图，并模拟火情蔓延扩散效果。

### 改动文件

| 文件 | 变更 |
|------|------|
| [wwwroot/project/js/heatmap.js](wwwroot/project/js/heatmap.js) | **新建** Canvas 2D 热力渲染引擎 + `SingleTileImageryProvider` 贴地显示 |
| [wwwroot/project/index.html](wwwroot/project/index.html) | 下拉菜单加"热力密度图" checkbox |
| [wwwroot/project/js/htmlmain.js](wwwroot/project/js/htmlmain.js) | 场景初始化后调用 `initHeatmap()` |

### 技术实现

- Canvas 2D 径向渐变圆叠加 → ImageData 颜色查找表映射 → `SingleTileImageryProvider` 贴地
- 颜色渐变：**蓝→青→绿→黄→橙→红**
- POI 权重：起火点(10) > 医院(6) > 消防站(5) > 政府(3) > 商场(2)
- **关键坑**：`SingleTileImageryProvider` 必须指定 `rectangle` 参数，否则图片默认覆盖全球看不到
- 火情蔓延模拟：基于真实 CSV 时间阈值线性插值，扩散半径随受影响建筑比例增长（已移除，改为纯 POI 密度热力）

## 五、CSV 编码修复

### 问题根因
编辑后的 `fire_poi_distance_all.csv` 以 **GBK(ANSI)** 编码保存，但程序默认 `Encoding.UTF8` 读取。更隐蔽的是 GBK 编码的字节在 UTF-8 严格验证下 `b3 a7`（厂）恰好是有效的 Latin-1 字符 `³§`，导致编码检测误判。

### 改动

| 文件 | 变更 |
|------|------|
| [new try world.csproj](new%20try%20world.csproj) | 添加 `System.Text.Encoding.CodePages` v10.0.9 NuGet |
| [Program.cs](Program.cs) | `Encoding.RegisterProvider(...)` 注册 GBK 支持 |
| [Program.cs](Program.cs) | 新增 `GuessEncoding()` 函数 |
| [Program.cs](Program.cs) | `ReadSpatialCsv()` 使用自动编码检测 |

### GuessEncoding 检测流程

```
有 BOM(EF BB BF) → UTF-8
全 ASCII → UTF-8
先试 GBK → 扫描 CJK 汉字(U+4E00~9FFF) → 有则返回 GBK
严格 UTF-8 验证(new UTF8Encoding(false, true)) → 通过则 UTF-8
回退 GBK
```
