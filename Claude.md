# CLAUDE.md — 超图杯 HTML+C# 项目

## 项目环境

- 开发工具：VS Code，已配置 Prettier 自动格式化
- 技术栈：ASP.NET Core 8.0 / C# / Razor(.cshtml) / HTML / CSS / JavaScript / SuperMap3D (iClient3D)
- 运行命令：`dotnet run`
- 静态资源根目录：`wwwroot`
- 严禁破坏 SuperMap3D 路径引用

## 文件读取与搜索规则

1. 每次默认最多读取 3 个相关文件。超过 3 个必须申请。
2. 允许通过关键字搜索定位代码（如 `grep "方法名" --include="*.cs"`），限定搜索范围在项目源码目录内。
3. 禁止无差别扫盘（`ls -R`、`find /`、遍历全盘目录）。
4. SuperMap3D 加载问题例外：允许同时检查 HTML 引用、C# 静态文件/CORS 中间件、wwwroot 文件存在性，共 3 个点位，不占用常规额度。
5. 不碰 `bin/`、`obj/`、`.vs/`、`.git/`、`*.dll`、`*.pdb`、`*.suo`。不输出连接字符串与密钥。

## 代码修改规范

1. 默认只替换错误片段。当方法/函数内错误 >3 处或逻辑需整体调整时，允许完整替换该方法/函数。
2. 输出完整可直接替换的代码，不省略命名空间、using 引用、关键逻辑。保证复制粘贴后无语法错误、可直接运行。
3. C# 遵循 .NET 官方命名规范，变量命名清晰；.cshtml 兼容 Razor 语法，不与后端逻辑冲突。
4. 每处修改标注：文件路径 + 行号范围、修改前代码、修改后代码、原因说明。
5. 多处修改按文件分组，保持原项目代码风格，不引入新框架或库。

## 允许直接操作

- 读取 ≤3 个相关文件（C# / .cshtml / HTML / JS / CSS）
- 通过关键字在源码目录内搜索定位代码
- 修改错误代码、路径、CORS/静态文件中间件配置
- 运行 `dotnet build`、`dotnet run`（仅用于短暂验证，非长期运行）
- 检查端口占用、检查文件存在性

## 必须申请的操作

- 读取 ≥4 个文件
- 无差别扫盘、批量遍历目录
- 项目重构、全面分析、初始化

## 回答风格

- 原因说清，修改对比直接给，不写项目总结、架构分析、长篇解释。
- 讲解适配 VS Code 环境，不使用 Visual Studio 专属功能。
- 报错优先分析底层原因，给出可直接落地的修复方案。
